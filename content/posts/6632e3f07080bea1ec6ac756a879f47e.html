随着 Coinbase 和 Cloudflare 将“402 Payment Required”转化为真正的按次付费基础设施，x402 正在走出 PING 热潮，迈向更广泛的应用。

如何加密

关键要点：

  • x402 实现了互联网按使用付费的功能。

  • 当前的势头由基础设施主导，主要由 Coinbase 和 Cloudflare 推动。

  • PING 是一个催化剂，但真正的故事是协议的采用，而不是代币。

  • 您可以通过启动一个端点并验证 402 → 支付 → 授权流程来快速测试。

X402 是一种在互联网上实现按使用付费的简单方法。当您访问一个付费的应用程序编程接口（API）或文件时，服务器会响应网络内置的“402 Payment Required”消息，指定价格——通常只是几美分的 USDC——以及支付的去向。

您从您的钱包发送链上支付，重复请求，服务器交付结果。没有账户、密码、API 密钥或月度计划——只是与该特定请求相关的一次性支付。

x402 的“第二波”

这个想法并不新鲜。402 状态码在 HTTP 中已经存在多年，但直到 2025 年，Coinbase 围绕它打包了一个明确的协议（“x402”），才有了实用的蓝图。该公司发布了文档和代码，并为开发者提供了一个托管网关。不久之后，Cloudflare 与 Coinbase 合作共同启动了 x402 基金会计划，正式化了标准，并为主流开发者工具带来了支持。

您可能第一次听说 x402 是因为一个名为 PING 的代币引起了关注。代币的热度消退了，但协议仍然存在，因为它解决了一个常见问题：按 API 调用、AI 推理或下载收费，而无需用户创建账户。

这种实用性，加上新的工具为AI 代理自动支付，推动了第二波关注实际使用而非价格图表的浪潮。

您知道吗？ X402 正成为 AI 代理自主支付的默认方式。Cloudflare 正在为其 Agents SDK 和 MCP 服务器添加原生 x402 支持。Coinbase 的新 Payments MCP 允许流行的大型语言模型持有钱包并完成请求，无需 API 密钥。

什么是 PING，它与 x402 有何关系？

PING是一种基于Base（Coinbase二层）的迷因币，也是首个通过x402流程公开铸造的代币，因此备受瞩目。早期买家无需网站注册，只需访问URL，即收到“402 Payment Required”消息，在链上用少量USDC付款、重试请求后获得PING。这可以视作x402按请求付费模式应用于铸造环节的现场演示。

该代币由X账号Ping.observer发布。公开报道与交易所均将PING归属于此账号。目前没有官方团队页面或白皮书，也无关于PING代币本身获得VC投资的可靠披露信息。

x402提供基础设施，而PING则作为首个大规模测试案例出现。该代币采用按付费铸造机制，对协议进行了压力测试，并突出展示了x402核心原则：每次请求收取极小链上费用，包括API调用、AI推理、文件下载或本例中的铸造，无需账户或API密钥。

初期涨跌过后，其持久影响不是代币价格，而是大量开发者和端点开始尝试并实验x402协议。

您知道吗？ PING 在 2025 年 10 月 25 日达到约 0.0776 美元的历史高点，然后在接下来的几天内回落。

如何尝试 x402（开发者快速入门）

1) 了解要点

X402 是一个简单的握手。您调用一个付费 URL，服务器回复“402 Payment Required”以及 USDC 的价格。您发送链上支付，然后再次调用 URL 并附上支付证明以获取结果。就是这样。

2) 选择您的设置

  • 托管：使用 Coinbase 的托管 x402 网关，带有仪表板和内置的交易了解（KYT）检查。它是快速概念验证的理想选择。

  • 自己动手（DIY）/规范：克隆开源 x402 参考实现，并在本地运行一个最小的卖家和买家，如果您想要完全控制。

3) 暴露一个付费端点

选择任何路径（例如，“/inference”）。当有人在未支付的情况下访问它时，返回一个“402”响应以及支付详情，包括金额、资产（USDC）、目标地址和到期时间。如果您可以使用“curl”触发该响应，您就正确地在使用 x402。

4) 完成一个付费请求

使用示例客户端或托管网关检测“402”，进行链上支付，然后重试请求。一旦支付确认，访问应自动更新，无需账户、API 密钥或 OAuth。

5) 可选：与 AI 代理测试

如果您与代理合作，启动模型上下文协议（MCP）示例。拦截器将检测“402”，从代理的钱包中进行支付并自动重新发出请求。这是确认代理到端点流的快速方法。

顶级提示：按照快速入门中的说明从测试网开始。一旦 402 → 支付 → 授权循环稳定，切换配置到主网。

风险、时间表和接下来要关注的内容

可能出错的地方

X402 仍然相对较新。规范和参考代码可能会继续演变，目前大多数实时设置使用 USDC。对单一托管网关或单一资产的过度依赖会带来供应商和资产集中风险。将代币叙述与协议进展分开也很重要。

治理进展关注点

请关注x402基金会正式成立相关细节，包括章程、成员名单及路线图，该事件将标志着协议从产品向标准转变。同时留意Cloudflare开发者生态系统（Agents SDK与MCP），因主流工具往往先于广泛采用而普及推广。

协议采纳信号

应重点观察那些能返回带有付款参数的真实端点，“先返回‘402’响应—再经链上付款解锁访问权限”，全程无需账户或API密钥。如果有更多快速入门教程、文档以及GitHub活跃度提升，则说明供给侧表现积极向好。

更广泛覆盖云服务商、内容分发网络(CDN)及除早期合作伙伴外更多代理框架，同时支持更多资产和网络，将使得x402愈加不可忽视。“智能体商业”集成持续推进，也很可能吸引此前未涉足加密领域的开发者参与其中。

如何保持信息同步

请关注主要渠道：Coinbase产品页、文档及GitHub获取协议最新动态，同时查阅Cloudflare博客与新闻稿了解基金会资讯和SDK支持情况。除这些官方渠道外的信息——尤其是有关代币讨论——都应视作背景噪音处理即可。
